# Security Policy

Do not open a public issue containing credentials, account identifiers, device
serials, Bluetooth addresses, precise location, raw authorization headers, or
unredacted captures.

Use GitHub private vulnerability reporting when available. Otherwise, publish
only a minimal description and request a private contact channel.

This is an independent documentation and research project, not Rokid's official
security-response channel. Vendor issues should also be reported through
Rokid's official support or developer contacts.
